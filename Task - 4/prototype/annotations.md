Annotated decisions for FitStart low-fidelity prototype

Welcome
- Keep the welcome screen minimal with a single clear CTA (Get Started) to reduce decision friction.

Onboarding — Activity Level
- Ask activity level early to personalize plan intensity and session count.
- Use three clear choices (Sedentary, Lightly active, Active) to keep it simple.

Onboarding — Preferences
- Allow selecting workout types via chips. These are optional and help tailor suggested workouts.

Profile Setup
- Collect only essential fields (name, age, height, weight) to reduce drop-off.
- Make fields optional where possible; defaults are used if left blank.

Goals
- Limit to up to 3 goals to prevent over-commitment; initial program targets top priorities.
- Present common goals with short labels for quick scanning.

Summary
- Show a brief recap so users can confirm before entering the app.

Interaction decisions
- Sequential flow keeps mental load low and ensures we collect the most impactful info first.
- Back buttons on each step let users correct choices without restarting.
- Annotations panel documents reasoning for product/design stakeholders.
